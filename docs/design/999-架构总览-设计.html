ntd (Now Task, Done) 技术架构全栈 AI 任务自动化平台 · 技术参考文档

ntd（Now Task, Done） 是一个全栈应用程序，允许用户创建 AI 驱动的待办事项，将其分配给各种代码执行 AI 代理（Claude Code、Codex、Pi 等），并通过 WebSocket 实时监控执行过程。该平台支持定时循环任务、飞书集成和云端同步。

Rust 1.81+ Axum 0.8 React 19 Vite 7 Ant Design 6 SQLite Tokio SeaORM 1 TypeScript 5.8 rustls

核心模块（16 个）

  • main.rs — 程序入口、CLI 解析、服务器启动、升级流程
  • lib.rs — 模块声明，Assets + NtdSkills 嵌入（rust-embed）
  • service_context.rs — 共享依赖容器（db、registry、tx、task_mgr、config）
  • config.rs — YAML 配置（~/.ntd/config.yaml）、校验、默认值、安全限制
  • handlers/ — HTTP 路由处理器（Axum 提取器 + AppState）
  • services/ — 业务逻辑（loop_runner、feishu_listener、message_debounce 等）
  • executor_service/ — 执行编排（pre-spawn → stages → completion）
  • adapters/ — AI 执行器接入（CodeExecutor trait，12+ 适配器）
  • db/ — 数据访问层（实体、迁移、查询构建器、辅助函数）
  • scheduler.rs — Cron 定时任务调度（tokio-cron-scheduler）
  • task_manager.rs — 运行中任务注册表 + 取消控制（RAII 守卫）
  • cli/ — CLI 客户端库（供 ntd npm 包消费者调用）
  • daemon/ — 系统服务管理（launchd/systemd/TaskScheduler）
  • feishu/ — 飞书/Lark 集成（SDK、WebSocket、消息编解码）
  • log_flusher.rs — 异步日志刷盘（定期写入磁盘）
  • sys.rs — FFI 边界层（SO_REUSEADDR、Unix 专属 libc 调用）

🧩 设计模式

1. 适配器模式 — AI 执行器接入

统一的 CodeExecutor trait 抽象了 12+ 种不同的 AI 编程代理。每种适配器实现相同的接口，无论底层协议是 stdio、事件流还是自定义协议。

adapters/mod.rs 定义了 EXECUTORS 静态数组，包含 ExecutorDef 元数据。ExecutorRegistry 通过名称/路径动态注册和解析执行器。

2. 广播频道 — 实时事件分发

tokio::sync::broadcast 容量可配（默认 10,000），将执行事件广播给所有连接的 WebSocket 客户端。环形缓冲区防止突发流量下的消息丢失。

通过 config.yaml 配置，带有安全限制：最小 100，软上限 1M，超过 100M 时崩溃拒绝。

3. 进程组管理 — 守护进程服务

跨平台守护进程管理：launchd（macOS）、systemd（Linux）、Task Scheduler（Windows）。共享路径解析在 daemon/common.rs，平台特定实现在子模块中。

Linux 支持分离式重新部署，实现零停机升级。

4. 日志刷写器 — 异步持久化

log_flusher.rs 实现定期的异步日志刷盘，将写入 I/O 与热执行路径解耦。使用 quick_cache 进行内存缓冲。

5. 服务上下文 — 依赖注入

ServiceContext 将 5 个 Arc<T> 依赖聚合为一个可克隆的结构体，贯穿整个应用，减少样板代码并确保一致的依赖共享。

字段：db、executor_registry、tx（广播）、task_manager、config（RwLock）。

6. 嵌入式资源 — RustEmbed

前端构建产物和 ntd-usage 技能文件通过 rust-embed 编译进二进制文件。实现单二进制分发，无需外部资源依赖。

Assets 结构体嵌入 ../frontend/dist/；NtdSkills 嵌入 ../ntd-skills/。

🚀 构建与部署

开发模式

npm run dev — Vite HMR 监听 :5173，代理 /api → localhost:8088

cargo run — 后端服务器，端口可配（默认 8088）

NTD_MODE=dev — 加载 config.dev.yaml

生产构建

npm run build — TypeScript + Vite 压缩打包

cargo build --release — LTO、单 codegen 单元、剥离符号

前端资源通过 rust-embed 嵌入二进制

交叉编译

musl 目标用于 Linux 静态构建

平台专属守护进程安装器（launchd/systemd/TaskScheduler）

条件编译：#[cfg(unix)]、#[cfg(windows)]

NPM 分发

发布为 @weibaohui/ntd

包含 CLI 二进制 + npm_utils 路径解析

ntd upgrade — 通过 npm + 守护进程重新部署自我升级

守护进程服务

ntd daemon install/start/stop/restart/status/uninstall

macOS：~/Library/LaunchAgents 中的 launchd plist

Linux：systemd 服务单元

Windows：任务计划程序注册

自动升级

ntd upgrade：npm install → 查找二进制 → daemon stop → uninstall → install --force → start

每次服务器启动时运行数据库迁移

执行器路径从 config.yaml 迁移至数据库
