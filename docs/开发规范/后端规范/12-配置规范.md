# 后端规范 12：配置规范

> 定义配置加载与环境管理规范。

---

## 1. 配置文件

| 环境 | 配置文件 | 数据库 | 端口 |
|------|---------|--------|------|
| 开发 | `~/.ntd/config.dev.yaml` | `~/.ntd/data.dev.db` | 18088 |
| 生产 | `~/.ntd/config.yaml` | `~/.ntd/data.db` | 8088 |

---

## 2. 配置结构体

```rust
// 使用 serde + config crate 加载 YAML 配置文件。
// 配置结构体在 backend/src/config.rs 中定义。
#[derive(Debug, Deserialize)]
pub struct AppConfig {
    pub server: ServerConfig,
    pub database: DatabaseConfig,
    pub executor: ExecutorConfig,
}

#[derive(Debug, Deserialize)]
pub struct ServerConfig {
    pub host: String,
    pub port: u16,
}

#[derive(Debug, Deserialize)]
pub struct DatabaseConfig {
    pub path: String,
}
```

---

```rust
    let config_path = if cfg!(debug_assertions) {

```

---

## 4. 禁止行为

- ❌ 在代码中硬编码端口、路径、密钥
- ❌ 在 handler/service 中直接读取环境变量（通过 AppConfig 统一注入）
- ❌ 配置文件提交到 Git（将示例文件 `config.yaml.example` 提交）
