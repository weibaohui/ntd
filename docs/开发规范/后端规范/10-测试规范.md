# 后端规范 10：测试规范

> 定义 ntd 后端测试编写规范。

---

## 1. 测试类型

| 类型 | 位置 | 说明 |
|------|------|------|
| 单元测试 | 模块内 `#[cfg(test)]` | 测试纯逻辑、工具函数、model 转换 |
| 集成测试 | `backend/tests/` | 测试 API 端点、数据库交互 |

---

## 2. 单元测试

每个公开函数都需要对应的单元测试，放在同一文件末尾的 `#[cfg(test)]` 模块中：

```rust
// 单元测试放在被测试代码所在文件的末尾，用 #[cfg(test)] 条件编译。
// 测试函数命名：test_<被测试函数名>_<场景>
// 覆盖：正常路径、边界条件、错误处理分支。
#[cfg(test)]
mod tests {
    use super::*;

    #[test]
    fn test_todo_response_from_model_contains_all_fields() {
        let model = TodoModel {
            id: 1,
            title: "Test".into(),
            status: StatusEnum::Active,
            created_at: Utc::now().naive_utc(),
        };
        let response = TodoResponse::from(model);
        assert_eq!(response.id, 1);
        assert_eq!(response.title, "Test");
    }
}
```

## 3. 集成测试

集成测试放在 `backend/tests/` 目录下，按文件拆分：

```rust
// 集成测试文件放在 backend/tests/ 目录下，每个文件一个测试目标。
// 使用 reqwest 发送 HTTP 请求验证 API 端点行为。
// 运行方式：cd backend && cargo test --test <文件名去掉 .rs>
//
// 例如：backend/tests/todo_api_test.rs
#[tokio::test]
async fn test_create_todo_returns_created() {
    // Arrange: 初始化测试数据库和测试服务器
    let app = test_app().await;

    // Act: 发送请求
    let response = app
        .post("/api/v1/todos")
        .json(&serde_json::json!({"title": "Test todo"}))
        .await;

    // Assert: 验证响应
    assert_eq!(response.status(), StatusCode::CREATED);
}
```

---

## 4. 测试数据库

集成测试使用独立的 SQLite 内存数据库或临时文件数据库：

```rust
// 测试数据库使用 :memory: 或临时文件，避免与开发/生产数据库冲突。
// 每个测试用例独立初始化，测试间互不干扰。
pub async fn test_db() -> DatabaseConnection {
    Database::connect("sqlite::memory:").await.unwrap()
}
```

---

## 5. 禁止行为

- ❌ 测试中访问生产/开发数据库
- ❌ 测试用例之间存在顺序依赖
- ❌ 测试中使用 `println!`（用 `assert!` / `assert_eq!` 做断言）
- ❌ 测试代码中包含 `unwrap()`（用 `assert!(result.is_ok())` 或 `?` 操作符）
