# 后端规范 02：项目分层

> 定义 ntd 后端的标准分层结构及各层的职责边界。

---

## 1. 目录结构

```text
backend/src/
├── handlers/          # Handler 层 — HTTP 请求处理
│   ├── mod.rs         # 路由注册聚合
│   ├── todo.rs        # Todo 相关资源
│   ├── execution.rs   # 执行记录
│   └── ...
├── services/          # Service 层 — 业务逻辑
│   ├── mod.rs
│   ├── todo_service.rs
│   └── ...
├── models/            # Model 层 — 数据结构
│   ├── todo.rs        # Entity / DTO / 枚举
│   └── ...
├── db/                # 数据库初始化 / 迁移
├── executor_service/  # 执行器适配（Claude Code 等）
├── adapters/          # 外部系统适配器
├── cli/               # CLI 命令处理
├── daemon/            # 守护进程管理
├── config.rs          # 配置加载
├── lib.rs             # 库入口
└── main.rs            # 入口
```

---

## 2. 各层职责

### 2.1 Handler 层（`handlers/`）

**职责范围**：
- 接收 HTTP 请求，提取参数
- 参数校验（格式、必填）
- 调用 Service 层方法
- 构造 HTTP Response（成功/错误）

**禁止**：
- ❌ 包含任何业务逻辑
- ❌ 直接操作数据库
- ❌ 使用 SeaORM Query 方法

### 2.2 Service 层（`services/`）

**职责范围**：
- 编排业务逻辑
- 调用 Repository（通过 Model 操作）
- 事务边界控制
- 调用外部服务/适配器

**禁止**：
- ❌ 引用 `axum` 类型（`Request`、`Response`、`Json`、`Extract` 等）
- ❌ 直接处理 HTTP 状态码

### 2.3 Model 层（`models/`）

**职责范围**：
- SeaORM Entity 定义
- DTO（Data Transfer Object）定义
- 业务枚举与常量
- `FromRow`、`Serialize`、`Deserialize` 派生

**类型区分**：

| 类型 | 说明 | 派生 |
|------|------|------|
| Entity | SeaORM 实体，对应表 | `#[derive(Clone, Debug, PartialEq, DeriveEntityModel)]` |
| DTO | API 请求/响应结构体 | `#[derive(Serialize, Deserialize)]` |
| Enum | 状态/类型枚举 | `#[derive(Debug, Clone, PartialEq)]` |

### 2.4 辅助模块

- `db/`：数据库连接池初始化、迁移管理
- `config.rs`：配置结构体，从 `~/.ntd/config.yaml` 加载
- `executor_service/`：执行器适配逻辑（Claude Code、OpenAI 等）
- `adapters/`：外部服务适配（飞书、Git 同步等）

---

## 3. 模块可见性规则

- `pub` 只暴露给上层调用（Handler 调 Service，Service 调 Repository）
- 同层之间调用用 `pub(crate)`
- 工具函数用 `pub(crate)` 或私有
