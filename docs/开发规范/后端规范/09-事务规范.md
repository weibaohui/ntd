# 后端规范 09：事务规范

> 定义事务边界管理与一致性保证规范。

---

## 1. 事务起点

事务在 Service 层开始和提交，Repository 层不感知事务：

```rust
// Service 层负责事务的开始和结束，Repository 方法接收事务连接作为参数。
// 这样 Repository 方法既可以独立使用（无事务），也可以在事务上下文中调用。
pub async fn create_todo_with_tags(
    &self,
    input: CreateTodoInput,
) -> Result<TodoResponse, AppError> {
    // 在 Service 方法中创建事务
    let txn = self.db.begin().await.map_err(AppError::from)?;

    // 将事务引用传给 Repository 方法
    let todo = self.todo_repo.insert(&txn, input.title).await?;
    for tag in input.tags {
        self.tag_repo.attach(&txn, todo.id, tag).await?;
    }

    // Service 层控制事务提交
    txn.commit().await.map_err(AppError::from)?;
    Ok(todo.into())
}
```

---

## 2. 事务传播

Repository 方法接收事务连接（`&DatabaseTransaction`）或普通连接（`&DatabaseConnection`）：

```rust
// Repository 方法使用 DatabaseConnection trait（DbConn 在 SeaORM 中是 DatabaseConnection 的别名），
// 这样传入事务连接或普通连接都可以工作。
impl TodoRepository {
    pub async fn insert(
        &self,
        conn: &DatabaseConnection,  // 可以是事务连接或普通连接
        title: String,
    ) -> Result<TodoModel, DbErr> {
        let active = todo::ActiveModel {
            title: Set(title),
            ..Default::default()
        };
        active.insert(conn).await
    }
}
```

---

## 3. 隐式事务

单表写操作可以使用 SeaORM 自带的事务（`Model::insert()` 默认在 auto-commit 模式）：

```rust
// 单次写入操作无需显式事务，SeaORM 的 auto-commit 足以保证原子性。
// 仅在跨表/跨 Repository 的操作需要显式事务。
let todo = TodoModel::insert(active_model).exec(&self.db).await?;
```

---

## 4. 事务隔离级别

SQLite 默认使用 `SERIALIZABLE` 隔离级别，多数场景不需要调整。

---

## 5. 回滚处理

- 使用 `?` 运算符：一旦失败自动向上传播，事务连接被 drop 时会自动回滚
- 不需要显式的 `rollback()` 调用（除非需要在回滚前执行清理逻辑）

```rust
// 需要清理的场景（回滚前释放外部资源）：
let txn = self.db.begin().await?;
match self.complex_operation(&txn).await {
    Ok(result) => {
        txn.commit().await?;
        Ok(result)
    }
    Err(e) => {
        // drop(txn) 会自动回滚，不需要显式调用 rollback
        Err(e)
    }
}
```

---

## 6. 禁止行为

- ❌ Handler 中控制事务开始/提交
- ❌ 事务跨越异步任务边界（`tokio::spawn` 内引用事务连接）
- ❌ 嵌套事务（SQLite 不支持）
