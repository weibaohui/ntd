# 后端规范 07：路由注册规范

> 定义 Axum 路由注册与中间件配置规范。

---

## 1. 路由组织

所有路由在 `handlers/mod.rs` 中按资源分组注册：

```rust
// 每个 handler 模块导出一个 router() 函数，返回独立的 Router。
// 在 handlers/mod.rs 中通过 .nest() 将子路由挂载到指定路径前缀下。
pub fn router() -> Router<AppContext> {
    Router::new()
        .nest("/api/v1/todos", todo::router())
        .nest("/api/v1/executions", execution::router())
        .nest("/api/v1/config", config::router())
        // ... 更多路由
}
```

---

## 2. 子路由定义

每个 handler 文件（如 `todo.rs`）中的 `router()` 函数：

```rust
// 每个资源模块内部使用 .nest("/{id}") 做路径层级嵌套，
// 这样在顶层统一路径前缀的情况下，路径定义在文件内是自描述的。
pub fn router() -> Router<AppContext> {
    Router::new()
        .route("/", get(list_todos).post(create_todo))
        .route("/{id}", get(get_todo).put(update_todo).delete(delete_todo))
}
```

---

## 3. 中间件

- 全局中间件（CORS、日志、超时）在 `main.rs` 或 `lib.rs` 中配置
- 路由组级别中间件（认证）在路由组上配置
- Handler 级别的中间件在具体路由上配置

```rust
// 路由组级别的中间件：在 group 的 router() 返回前调用 .layer() 或 .route_layer()。
// 例如认证中间件只对特定资源组生效，而不是全局。
pub fn router() -> Router<AppContext> {
    Router::new()
        .route("/", get(list))
        .layer(AuthLayer::new())
}
```

## 4. OpenAPI 文档

使用 `utoipa` 或 `axum-openapi` 自动生成 OpenAPI 规范。

Handler 函数上的文档注释是生成 API 文档的主要来源：

```rust
/// List all todos with pagination and filtering.
///
/// Supports filtering by status and keyword search.
/// Returns paginated results ordered by created_at desc.
#[utoipa::path(
    get,
    path = "/api/v1/todos",
    params(ListTodoParams),
    responses(
        (status = 200, description = "Todo list", body = Vec<TodoResponse>)
    )
)]
pub async fn list_todos(/* ... */) -> Result<Json<Vec<TodoResponse>>, AppError> {
    // ...
}
```

---

## 5. 禁止行为

- ❌ 在 Handler 外部（Service/Model）使用 `axum::Router` 或路由宏
- ❌ 在 `main.rs` 中直接定义路由（集中在 `mod.rs` 管理）
- ❌ 路径硬编码字符串分散在各 handler 文件中（集中在 `router()` 函数）
