# 后端规范 05：错误处理规范

> 定义 ntd 后端的统一错误处理机制。

---

## 1. AppError 枚举

所有业务错误统一使用 `AppError` 枚举，实现 `IntoResponse` 以直接作为 Axum Handler 返回值：

```rust
// AppError 覆盖所有业务场景的错误类型，每个变体关联一个 HTTP 状态码。
// 通过实现 IntoResponse，Handler 可直接返回 Result<T, AppError>。
#[derive(Debug, thiserror::Error)]
pub enum AppError {
    #[error("Not found: {0}")]
    NotFound(String),

    #[error("Bad request: {0}")]
    BadRequest(String),

    #[error("Forbidden: {0}")]
    Forbidden(String),

    #[error("Internal error: {0}")]
    Internal(String),

    #[error("Conflict: {0}")]
    Conflict(String),
}
```

---

## 2. IntoResponse 实现

```rust
// 将 AppError 转换为 HTTP Response：
// - 错误枚举变体 → 对应的 HTTP 状态码
// - 错误消息 → JSON body { "error": "..." }
// - 统一记录 error 级别日志
impl IntoResponse for AppError {
    fn into_response(self) -> Response {
        let status = match &self {
            AppError::NotFound(_) => StatusCode::NOT_FOUND,
            AppError::BadRequest(_) => StatusCode::BAD_REQUEST,
            AppError::Forbidden(_) => StatusCode::FORBIDDEN,
            AppError::Internal(_) => StatusCode::INTERNAL_SERVER_ERROR,
            AppError::Conflict(_) => StatusCode::CONFLICT,
        };
        let body = Json(serde_json::json!({ "error": self.to_string() }));
        (status, body).into_response()
    }
}
```

---

## 3. 错误转换

使用 `From` trait 或 `map_err` 将底层错误转换为 `AppError`：

```rust
// 将 SeaORM 的 DbErr 或标准库的 io::Error 等转换为 AppError，
// 确保 ? 运算符在各层之间工作一致。
impl From<DbErr> for AppError {
    fn from(err: DbErr) -> Self {
        AppError::Internal(err.to_string())
    }
}
```

---

## 4. 错误传播规则

| 层次 | 错误类型 | 传播方式 |
|------|---------|---------|
| Repository | `DbErr` / `AppError` | 通过 `?` 向上传播 |
| Service | `AppError` | 包装为合适的 `AppError` 变体 |
| Handler | `AppError` | 由 `IntoResponse` 自动转为 HTTP Response |

---

## 5. 禁止行为

- ❌ 在 Handler 中使用 `unwrap()` / `expect()` 处理错误
- ❌ 在 Handler 中手动匹配错误类型并设置 StatusCode
- ❌ 直接返回 `(StatusCode, String)` 作为 Response（用 AppError）
- ❌ 在错误消息中包含敏感信息（数据库细节、堆栈路径）
