# 前端规范 11：测试规范

> 定义前端测试编写规范。

---

## 1. 测试工具

| 类型 | 工具 | 用途 |
|------|------|------|
| 单元测试 | Vitest | 纯函数、Hooks、组件渲染 |
| 集成/功能测试 | Playwright | 页面交互、用户流程 |

---

## 2. 单元测试（Vitest）

测试文件与源文件放在同一目录，命名 `<文件名>.test.ts` 或 `<文件名>.test.tsx`：

```tsx
// TodoCard.test.tsx
// 单元测试测试组件渲染和交互，不依赖真实后端。
import { render, screen } from '@testing-library/react';
import { describe, it, expect } from 'vitest';
import { TodoCard } from './TodoCard';

describe('TodoCard', () => {
  it('renders todo title', () => {
    render(<TodoCard todo={{ id: '1', title: '测试' }} onClick={() => {}} />);
    // 通过 screen 查询渲染内容，而不是依赖 className 或 data-testid
    expect(screen.getByText('测试')).toBeDefined();
  });
});
```

---

## 3. Playwright 功能测试

Playwright 测试文件放在 `frontend/tests/` 目录下：

```ts
// frontend/tests/check_todo.spec.ts
// 功能测试模拟完整用户操作路径，验证 UI 交互效果。
import { test, expect } from '@playwright/test';

test('用户创建 Todo 流程', async ({ page }) => {
  // 1. 打开页面
  await page.goto('http://localhost:18088');

  // 2. 输入标题并提交
  await page.fill('[data-testid="todo-input"]', '新任务');
  await page.click('[data-testid="submit-btn"]');

  // 3. 验证新增项出现
  await expect(page.locator('.todo-item')).toContainText(['新任务']);
});
```

---

## 4. 注意事项

- 所有测试必须**独立可运行**，不依赖外部状态
- 测试之间不得互相依赖或共享可变状态
- Playwright 测试运行前确保开发服务器已启动（`make dev`）
- 截图证据发布到 PR/Issue 评论，**不提交到 Git 仓库**
