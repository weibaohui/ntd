# 前端规范 13：禁止清单

> 明确定义前端代码中**严禁出现**的代码模式。

---

## 代码模式禁止

| # | 禁止项 | 原因 | 替代方案 |
|---|--------|------|---------|
| 1 | `any` 类型 | 丧失类型安全 | `unknown` + 类型守卫 |
| 2 | `// @ts-ignore` / `// @ts-expect-error` | 跳过类型检查 | 正确定义类型 |
| 3 | `var` 声明 | 作用域问题 | `const` / `let` |
| 4 | 直接修改 props 或 state | 不可变性违反 | `setState` / 不可变更新 |
| 5 | `dangerouslySetInnerHTML` | XSS 风险 | 安全渲染（sanitize） |
| 6 | 空 `catch` 块 | 吞没错误 | 至少记录或提示 |
| 7 | 组件内直接使用 `fetch(url)` | 不一致 | `api/` 模块统一调用 |
| 8 | `document.title = ...` 等 DOM 直接操作 | React 与 DOM 不一致 | React / ref 管理 DOM |
| 9 | `setInterval` 未清理 | 内存泄漏 | `useRef + useEffect` 清理 |
| 10 | 硬编码颜色/间距值 | 主题不一致 | theme token / 设计变量 |

---

## 架构禁止

| # | 禁止项 | 原因 | 替代方案 |
|---|--------|------|---------|
| 11 | React Class 组件 | Hooks 时代模式过时 | 函数组件 + Hooks |
| 12 | `../` 回退超过 2 级 | 导入路径混乱 | `@/` 路径别名 |
| 13 | 全局 CSS 覆盖 Ant Design 组件样式 | 样式冲突 | `ConfigProvider.theme` + CSS Module |
| 14 | 大文件（单组件 > 300 行） | 难以维护 | 拆分为子组件和 Hook |
| 15 | 在组件内直接 import 图片二进制 | 编译产物膨胀 | 放在 public/ 或使用 URL |

---
