# 前端规范 10：导入规范

> 定义模块导入顺序与路径规范。

---

## 1. 导入顺序

按以下顺序分块导入，块之间空一行：

```tsx
// 1. 标准库 / 第三方依赖
// 外部依赖放在最前面，便于读者一眼看出项目的依赖关系。
import React, { useState, useEffect } from 'react';
import { Card, Button, Table } from 'antd';
import { useNavigate } from 'react-router-dom';

// 2. 绝对路径导入（@/）
// 跨目录的内部模块使用 @/ 别名，避免相对路径回退。
import { useTodos } from '@/hooks/useTodos';
import { TodoCard } from '@/components/todo/TodoCard';
import { fetchTodos } from '@/api/todos';
import type { Todo, TodoStatus } from '@/types/todo';

// 3. 相对路径导入（仅限同目录）
// 仅当文件在同一目录时使用相对导入。
import { helpers } from './helpers';
import styles from './TodoCard.module.css';
```

---

## 2. 路径别名规则

**强制要求**：`frontend/src` 内的跨目录导入统一使用 `@/` 别名：

```tsx
// ✅ 正确
import { useTheme } from '@/hooks/useTheme';
import { API_BASE } from '@/constants';

// ❌ 错误（跨目录回退）
import { useTheme } from '../../hooks/useTheme';
import { API_BASE } from '../constants';
```

---

## 3. 类型导入

使用 `import type` 语法区分类型导入：

```tsx
// ✅ 类型导入用 import type，运行时导入用 import
// 这能让打包器识别出可 tree-shake 的类型引用。
import type { Todo, PaginatedResponse } from '@/types/todo';
import { fetchTodos } from '@/api/todos';
```

---

## 4. 禁止行为

- ❌ 使用 `import * as X from '...'` 导入整个模块（除非不可避免）
- ❌ 在非类型文件中使用相对路径跨多级目录导入（超过 `../` 两级）
- ❌ 导入未使用的变量或类型
