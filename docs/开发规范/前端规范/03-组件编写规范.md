# 前端规范 03：组件编写规范

> 定义 React 组件的编写规范。

---

## 1. 组件类型

优先使用函数组件 + Hooks，不使用 class 组件：

```tsx
// ✅ 函数组件 + 类型标注
// 使用 TypeScript 定义 Props 类型，利用 React.FC 泛型或直接标注。
const TodoCard: React.FC<{ todo: Todo; onClick: (id: string) => void }> = ({
  todo,
  onClick,
}) => {
  // ...
};
```

---

## 2. Props 类型定义

```tsx
// Props 类型定义在组件文件顶部，命名规范：<ComponentName>Props。
// 使用 interface 而非 type（interface 的可扩展性和合并能力更好）。
interface TodoCardProps {
  /** Todo 数据对象 */
  todo: Todo;
  /** 点击卡片回调，参数为 Todo ID */
  onClick: (id: string) => void;
  /** 是否处于加载态 */
  loading?: boolean;
}
```

---

## 3. 组件职责

- 一个组件只做一件事
- 组件不应该超过 150 行（含 JSX）
- 复杂 UI 拆分为多个小组件组合

---

## 4. JSX 规范

```tsx
// ✅ 清晰的 JSX 结构：
// - 使用语义化的 JSX 标签
// - 条件渲染使用 && 或三元表达式，避免复杂逻辑
// - 复杂条件抽离为变量或函数
const TodoCard: React.FC<TodoCardProps> = ({ todo, onClick, loading }) => {
  // 将条件判断提为变量，保持 JSX 干净
  const isOverdue = todo.dueDate && new Date(todo.dueDate) < new Date();

  return (
    // 注意：Card 的 onClick 是 antd Card 组件的回调，不是 HTML 原生事件
    <Card
      hoverable
      loading={loading}
      onClick={() => onClick(todo.id)}
      className={isOverdue ? styles.overdue : undefined}
    >
      <Card.Meta title={todo.title} description={todo.description} />
    </Card>
  );
};
```

---

## 5. Hooks 规范

```tsx
// 自定义 Hook 以 useXxx 命名，放置于 hooks/ 目录。
// Hook 返回值使用对象形式（方便调用方解构且不受顺序影响）。
export function useTodoList(initialFilter?: FilterParams) {
  const [todos, setTodos] = useState<Todo[]>([]);
  const [loading, setLoading] = useState(false);

  // 使用 useEffect 注意依赖数组完备性，必要时用 useRef 消除闭包陷阱
  useEffect(() => {
    fetchTodos();
  }, []); // eslint-disable-line react-hooks/exhaustive-deps

  return { todos, loading, refetch: fetchTodos };
}
```
