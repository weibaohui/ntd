# 前端规范 09：命名规范

> 定义变量、组件、文件命名规则。

---

## 1. 命名规则总表

| 元素 | 规范 | 示例 |
|------|------|------|
| 组件名 | PascalCase | `TodoCard`、`SearchBar` |
| 文件名（组件） | PascalCase | `TodoCard.tsx` |
| 文件名（非组件） | camelCase | `useTodos.ts`、`todoApi.ts` |
| 变量名 | camelCase | `todoList`、`isLoading` |
| 常量 | SCREAMING_SNAKE | `MAX_PAGE_SIZE` |
| 枚举 | PascalCase | `TodoStatus` |
| 枚举值 | PascalCase | `Active`、`Completed` |
| 类型/接口 | PascalCase | `TodoProps`、`UserInfo` |
| 函数名 | camelCase | `formatDate`、`fetchTodo` |
| 自定义 Hook | camelCase + use 前缀 | `useTodoList` |
| CSS 类名 | kebab-case | `todo-card`、`search-bar` |
| CSS Module | camelCase | `styles.todoCard` |

---

## 2. 布尔变量

- 使用 `is`、`has`、`should`、`can` 前缀

```ts
// ✅ 明确的布尔命名
const isLoading = true;
const hasPermission = false;
const shouldShowModal = isLoading || hasPermission;

// ❌ 不明确的命名
const load = true;
const flag = false;
```

---

## 3. 事件回调

- 使用 `on` 前缀表示回调 prop
- 使用 `handle` 前缀表示事件处理函数

```tsx
// ✅ 清晰的回调命名
interface TodoCardProps {
  onDelete: (id: string) => void;
  onStatusChange: (id: string, status: TodoStatus) => void;
}

const handleDelete = async () => {
  await deleteTodo(todo.id);
  onDelete(todo.id);
};
```
