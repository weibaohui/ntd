# 前端规范 12：最佳实践

> 总结常见场景的推荐做法。

---

## 1. 条件渲染

```tsx
// ✅ 简单的条件用 &&
{isLoading && <Spin />}

// ✅ 2 分支用三元表达式
{isError ? <Alert message={error} type="error" /> : <TodoList />}

// ❌ 复杂逻辑放在 JSX 内
// 将条件逻辑提为变量或函数，JSX 只做渲染。
// 用 renderContent() 或 computed 变量替代内联逻辑。
const renderContent = () => {
  if (isLoading) return <Skeleton />;
  if (isError) return <Result status="error" />;
  if (isEmpty) return <Empty description="暂无数据" />;
  return <TodoTable dataSource={todos} />;
};
```

## 2. 列表渲染

```tsx
// ✅ 始终使用 key（唯一且稳定）
// key 使用数据的 id 而非索引，避免 React 的重渲染 bug。
{todos.map((todo) => (
  <TodoCard key={todo.id} todo={todo} />
))}
```

---

## 3. useEffect 清理

```tsx
// ✅ 使用 clean-up 函数处理竞态
// 当组件卸载或依赖变化时取消正在进行的异步操作，
// 避免"在已卸载的组件上 setState"的 warning。
useEffect(() => {
  let cancelled = false;
  fetchData().then((data) => {
    if (!cancelled) {
      setData(data);
    }
  });
  return () => {
    cancelled = true;
  };
}, [deps]);
```

## 4. 表单

```tsx
// 使用 Ant Design 的 Form 组件，并在业务组件中保持表单纯函数：
// - 校验规则使用 Form.Item 的 rules 属性
// - 提交方法从 form 实例获取 values
// - 避免在表单组件中通过 useState 跟踪每个字段值
const [form] = Form.useForm();

const handleSubmit = async (values: CreateTodoValues) => {
  await createTodo(values);
  message.success('创建成功');
  form.resetFields();
};

<Form form={form} onFinish={handleSubmit}>
  <Form.Item name="title" rules={[{ required: true, message: '请输入标题' }]}>
    <Input placeholder="Todo 标题" />
  </Form.Item>
</Form>
```

## 5. 性能

- `useMemo` 和 `useCallback` 仅在明确观察到性能问题时使用（不预先优化）
- 大数据量列表使用 `react-window` 或虚拟滚动
- 频繁变化的 Context 值用 `useMemo` 包裹
