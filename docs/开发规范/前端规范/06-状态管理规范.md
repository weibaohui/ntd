# 前端规范 06：状态管理规范

> 定义前端状态管理策略。

---

## 1. 状态分类

| 状态类型 | 管理方式 | 示例 |
|---------|---------|------|
| 组件局部状态 | `useState` / `useReducer` | 表单输入、展开/折叠 |
| 跨组件状态 | React Context | 主题、用户登录 |
| 服务端状态 | 每次请求重新获取 | Todo 列表、执行记录 |
| URL 状态 | `useSearchParams` | 查询参数、分页页码 |

---

## 2. 服务端数据

ntd 前端采用"每次请求重新拉取"的策略，不需要全局状态管理库（Redux/Zustand）：

```tsx
// 服务端数据通过自定义 Hook 封装，在组件中按需调用。
// 不使用全局 Store 缓存服务端数据，避免缓存一致性问题。
function useTodoDetail(id: string) {
  const [todo, setTodo] = useState<Todo | null>(null);
  const [loading, setLoading] = useState(true);

  useEffect(() => {
    // 当 id 变化时重新拉取，同时处理竞态
    let cancelled = false;
    fetchTodo(id).then((data) => {
      if (!cancelled) {
        setTodo(data);
        setLoading(false);
      }
    });
    return () => { cancelled = true; };
  }, [id]);

  return { todo, loading };
}
```

## 3. Context 使用

- Context 仅用于真正需要跨组件共享的状态（主题、用户信息）
- Context 值不应频繁变化（导致所有消费者重渲染）
- 拆分多个 context 而非一个大而全的 context

```tsx
// ✅ 将不相关的状态拆分到不同 Context 中
// ThemeContext 只在切换主题时更新，不随用户操作频繁变化。
// UserContext 在登录/登出时更新，不与主题数据耦合。
export const ThemeContext = createContext<ThemeConfig>(defaultTheme);
export const UserContext = createContext<User | null>(null);
```
