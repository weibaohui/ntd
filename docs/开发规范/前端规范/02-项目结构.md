# 前端规范 02：项目结构

> 定义前端目录结构与文件组织规范。

---

## 1. 目录结构

```text
frontend/src/
├── api/              # API 调用函数（封装 fetch/axios）
│   ├── client.ts     # HTTP 客户端实例
│   ├── todos.ts      # Todo API
│   └── ...
├── components/       # 可复用 UI 组件
│   ├── common/       # 通用组件（Layout、Button、Modal）
│   ├── todo/         # Todo 领域组件
│   └── ...
├── hooks/            # 自定义 React Hooks
│   ├── useTodos.ts
│   └── ...
├── pages/            # 页面级组件（路由入口）
│   ├── TodoList.tsx
│   ├── TodoDetail.tsx
│   └── ...
├── types/            # TypeScript 类型定义
│   ├── todo.ts
│   └── ...
├── themes/           # 主题配置
├── utils/            # 工具函数
├── constants.ts      # 项目常量
├── App.tsx           # 应用根组件
└── main.tsx          # 入口文件
```

---

## 2. 文件组织原则

| 层次 | 职责 | 示例 |
|------|------|------|
| `pages/` | 路由入口，组合领域组件 | `TodoListPage.tsx` |
| `components/` | 可复用 UI 组件 | `TodoCard.tsx`、`SearchBar.tsx` |
| `hooks/` | 状态逻辑抽离 | `useTodoList.ts` |
| `api/` | HTTP 请求封装 | `client.ts`、`todoApi.ts` |
| `types/` | 纯类型定义 | `todo.ts`、`common.ts` |
| `utils/` | 纯函数工具 | `formatDate.ts` |

---

## 3. 文件命名

- React 组件：`PascalCase.tsx`（如 `TodoCard.tsx`）
- Hook：`useXxx.ts`（如 `useTodos.ts`）
- 工具函数：`camelCase.ts`（如 `formatDate.ts`）
- 类型文件：`camelCase.ts`（如 `todo.ts`）
- API 模块：`camelCase.ts`（如 `todoApi.ts`）
- 样式：`camelCase.css` 或直接 `inline / CSS-in-JS`

---

## 4. 导入规范

**强制要求**：跨目录导入统一使用 `@/` 绝对路径别名，禁止 `../`、`../../` 回退。

```tsx
// ✅ 正确
import { useTodos } from '@/hooks/useTodos';
import { TodoCard } from '@/components/todo/TodoCard';

// ❌ 错误
import { useTodos } from '../../hooks/useTodos';
import { TodoCard } from '../todo/TodoCard';
```

例外：同目录内的短相对导入可以保留，如 `./constants`、`./helpers`。
