# 前端规范 08：类型定义规范

> 定义 TypeScript 类型编写规范。

---

## 1. 类型组织

```text
frontend/src/types/
```

---

## 2. 接口 vs 类型别名

- 优先使用 `interface`（可扩展、性能更好）
- 联合类型、交叉类型、工具类型用 `type`

```ts
// ✅ Props 和 State 形状使用 interface
// interface 可以重复声明合并，更适合公共 API 的形状定义。
export interface Todo {
  id: string;
  title: string;
  status: TodoStatus;
  createdAt: string;
}

// ✅ 联合类型使用 type
// type 更适合表达组合类型。
export type TodoStatus = 'active' | 'completed' | 'archived';

// ✅ 函数类型签名使用 type
export type FetchTodosFn = (params: ListParams) => Promise<PaginatedResponse<Todo>>;
```

---

## 3. 命名风格

```ts
// 接口：不携带 I 前缀
// ❌ ITodoProps — 不要使用 I 前缀
// ✅ TodoProps — 直接使用名词
export interface TodoProps {
  todo: Todo;
}

// 枚举：PascalCase，值使用 PascalCase
export enum TodoStatus {
  Active = 'active',
  Completed = 'completed',
  Archived = 'archived',
}
```

---

## 4. 禁止 any

```ts
// ❌ 禁止
const data: any = await fetchTodo(id);

// ✅ 始终指定类型
const data: Todo = await fetchTodo(id);

// ✅ 不确定时使用 unknown + 类型守卫
const data: unknown = await fetchTodo(id);
if (isTodo(data)) {
  console.log(data.title);
}
```
