# 1. 背景（Why）

当前设置页的「备份与恢复」标签下有三组自动备份功能：

- 数据库备份
- Todo 备份
- Skill 备份

这三组功能的默认开启状态均为「关闭」（`false`），需要用户手动进入设置页逐一开启。新用户首次部署后不会主动去做这件事，导致核心数据缺乏自动保护。

# 2. 目标（What，必须可验证）

- [ ] 数据库自动备份默认改为「开启」（Cron: 每天凌晨 3 点，保留 30 份）
- [ ] Todo 自动备份默认改为「开启」（Cron: 每天凌晨 4 点，保留 30 份）
- [ ] Skill 自动备份默认改为「开启」（Cron: 每天凌晨 5 点，保留 30 份）
- [ ] 新增配置文件的用户自动获得以上默认值
- [ ] 已有配置文件用户不受影响（不覆盖已写入的显式值）

# 3. 非目标（Out of scope）

- 不改自动更新（`auto_update_enabled`）默认值
- 不改内置资源自动同步（`bundled_source.auto_sync_enabled`）默认值
- 不改 AI 使用统计自动归档（`auto_usage_stats_enabled`）默认值
- 不改日志清理保留天数（已默认 30 天，不动）
- 不处理已有配置文件的迁移逻辑
- 不涉及前端 UI 变更

# 4. 验收标准

1. 全新启动（配置文件不存在时），`auto_backup_enabled` / `auto_todo_backup_enabled` / `auto_skill_backup_enabled` 均为 `true`
2. `cargo test` 全部通过
3. `cargo clippy -- -D warnings` 零告警
