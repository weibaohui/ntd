# 实现总结：三组自动备份默认值改为开启

## 需求对应

对应已确认的设计方案：`docs/design/024-自动备份默认开启-设计.md`

## 改动内容

### 文件变更

| 文件 | 改动 | 说明 |
|------|------|------|
| `backend/src/config.rs:88` | `enabled: false` → `enabled: true` | `BackupScheduleDefaults::new()` 中默认 enabled 改为开启 |
| `backend/src/config.rs:88` | 更新注释 | 反映新的默认值意图 |
| `backend/src/config.rs:549-550` | `assert!(!enabled)` → `assert!(enabled)` | 同步更新测试断言 |
| `backend/src/config.rs:569-571` | `assert!(!s.enabled)` → `assert!(s.enabled)` | 同步更新单测断言 |

### 涉及字段（三组，全部同时变更）

- `auto_backup_enabled` — 数据库自动备份（Cron: 每天凌晨 3 点）
- `auto_todo_backup_enabled` — Todo 自动备份（Cron: 每天凌晨 4 点）
- `auto_skill_backup_enabled` — Skill 自动备份（Cron: 每天凌晨 5 点）

### 关键实现点

1. **一处改动，三组生效**：`BackupScheduleDefaults` 是三组备份 enabled 默认值的唯一来源，通过 `Default for Config` 的装配逻辑一次性应用到数据库/Todo/Skill 三组配置字段。
2. **对已有配置用户无影响**：`serde(default)` 只在配置文件不存在时生效；已有 `config.yaml` 中如果显式写了 `auto_backup_enabled: false` 则不受影响。
3. **启动时自动注册**：`main.rs:599-620` 按 `auto_*_backup_enabled` 为 `true` 时注册定时任务，默认值改为 true 后启动即自动开启。

## 验证结果

- `cargo clippy --all-targets -- -D warnings`：零告警 ✅
- `cargo test`：全部通过 ✅（含 `test_backup_schedule_defaults_new`、`test_default_auto_backup_and_sync_fields_match_helpers` 等配置相关单测）

## 已知限制

- 已存在的配置文件（已有用户）不受影响，不会自动把 false 改为 true。如果需要"迁移现有用户的旧配置"，需要额外编写迁移逻辑，但本次不包含该范围。
